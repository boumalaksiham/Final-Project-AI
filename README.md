# Multi-Agent Scientific Paper Analysis System

> Automatically analyze scientific papers and generate structured triage reports — in seconds.

A multi-agent AI system built for **paper triage**: helping researchers decide whether a paper is worth reading *before* investing time in it. Six specialized agents each handle one distinct task, and a Coordinator synthesizes their outputs into a unified structured report.

---

## What It Does

Given a paper (PDF, arXiv ID, or PubMed ID), the system generates a report containing:

- **TL;DR** — a single sentence capturing the paper's contribution
- **Relevance score** — 0–100 match against your stated research topic
- **Summary** — abstractive summary generated by BART
- **Citation graph** — directed knowledge graph of all references
- **Methodology** — datasets used, evaluation metrics, model architectures
- **Limitations** — author-stated limitations extracted from the paper
- **Agent summary table** — status and key output of every agent

---

## System Architecture

```
INPUT: PDF / arXiv ID / PubMed ID
         ↓
  paper_loader.py (PyMuPDF + arXiv API + NCBI API)
         ↓
┌─────────────────────────────────────────────┐
│  Agent 1: Summarization    (BART)            │
│  Agent 2: Citation         (NetworkX)        │
│  Agent 3: Methodology      (spaCy NER)       │
│  Agent 4: Critical         (Pattern match)   │
│  Agent 5: Relevance/TL;DR  (Keyword scoring) │
└─────────────────────────────────────────────┘
         ↓
  Agent 6: Coordinator (synthesis)
         ↓
OUTPUT: Markdown report + Citation graph PNG + raw JSON
```

### Agent Details

| Agent | Task | Technique |
|-------|------|-----------|
| **1 — Summarization** | Abstractive summary | BART (`facebook/bart-large-cnn`), section-aware chunking, PDF artifact cleaning |
| **2 — Citation Analysis** | Extract references, build knowledge graph | Regex (3 citation formats: numbered, abbreviated-key, author-year), NetworkX, arXiv API |
| **3 — Methodology Extractor** | Identify datasets, metrics, architectures | spaCy NER + curated keyword matching |
| **4 — Critical Analysis** | Extract limitations and future work | Pattern matching on isolated sections with TOC-skip heuristic |
| **5 — Relevance & TL;DR** | Relevance score + single-sentence TL;DR | Keyword matching against user topic, structured sentence synthesis |
| **6 — Coordinator** | Synthesize all outputs into unified report | Multi-agent output merging, Markdown + JSON |

## Setup

### Requirements

- Python 3.10+
- macOS, Linux, or Windows

### Install

```bash
pip install -r requirements.txt
python -m spacy download en_core_web_sm
```

### Known Issue — Apple Silicon (MPS backend)

BART produces empty output on Apple Silicon's MPS backend due to a known PyTorch/Transformers bug. This is handled automatically in the code by forcing CPU inference (`device=-1`). No action needed.

## Usage

### Run on built-in demo (no download needed)

```bash
python main.py --demo
```

### Analyze any arXiv paper by ID

```bash
python main.py --arxiv 1706.03762          # Attention Is All You Need
python main.py --arxiv 2005.14165          # GPT-3
python main.py --arxiv 1810.04805          # BERT
python main.py --arxiv 1512.03385          # ResNet
```

### Analyze with a research topic for relevance scoring

```bash
python main.py --arxiv 1706.03762 --topic "attention mechanisms transformer NLP"
```

The `--topic` flag enables the Relevance Agent to compute a 0–100 relevance score and a verdict:
- **≥ 70** — Highly relevant — strongly recommend reading
- **40–69** — Possibly relevant — skim the abstract and introduction
- **15–39** — Loosely related — may contain useful references
- **< 15** — Likely not relevant to your research topic

### Analyze a local PDF

```bash
python main.py --pdf path/to/paper.pdf
```

### Fetch a PubMed paper by PMID (biomedical papers)

```bash
python main.py --pubmed 33278872
```

### Run evaluation (ROUGE + extraction F1)

```bash
python evaluate.py 1706.03762
python evaluate.py 2005.14165
```

### Run ablation study

```bash
python ablation.py 1706.03762
python ablation.py 2005.14165
```

### Run single-model baseline comparison

```bash
python baseline.py 1706.03762
python baseline.py 2005.14165
```

### Build cross-paper citation graph (shared references across papers)

```bash
python cross_paper_graph.py --arxiv 1706.03762 2005.14165 1810.04805 1512.03385
```

### Run qualitative evaluation on unseen papers

```bash
# Step 1 — generate reports for 5 unseen papers
python qualitative_eval.py --arxiv 2106.09685 2203.02155 2302.13971 2210.11610 2005.11401

# Step 2 — score each report interactively (guided rubric)
python qualitative_eval.py --score

# Print saved scores summary
python qualitative_eval.py --summary
```

| Paper | ROUGE-1 | ROUGE-2 | Dataset F1 | Metric F1 | Model F1 | Citations |

| Attention Is All You Need | 0.458 | 0.192 | 0.800 | 0.444 | **1.000** | 30 |

| BERT | 0.374 | 0.211 | 0.800 | 0.444 | 0.875 | 30 |
| ResNet | 0.347 | 0.183 | 0.800 | 0.500 | 0.333 | 30 |
| **Average** | **0.431** | **0.251** | **0.850** | **0.597** | **0.802** | 30 |

### Multi-Agent vs. Single-Model BART Baseline

| Metric | Baseline | Multi-Agent | Improvement |
|--------|----------|-------------|-------------|
| ROUGE-1 | 0.185 | 0.502 | **+0.317** |
| ROUGE-2 | 0.114 | 0.306 | **+0.192** |

| Metric F1 | 0.444 | 0.722 | **+0.278** |
| Model F1 | 0.554 | 1.000 | **+0.446** |

### Ablation Study — GPT-3 Paper

Removing any single agent causes complete failure in its dimension:

| Agent Removed | Effect |
|---------------|--------|
| Summarization | ROUGE-1 → 0.000, ROUGE-2 → 0.000 |
| Citation | References extracted → 0 |
| Methodology | Dataset F1, Metric F1, Model F1 → 0.000 |
| Critical | Limitations count → 0 |

### Cross-Paper Citation Graph — 4 Papers

- **15 shared references** identified across the 4 quantitative papers
- **4 direct citation links** between papers detected (GPT-3 → BERT, GPT-3 → ResNet, BERT → ResNet, Attention → ResNet)
- Bahdanau et al. 2015 and Chung et al. 2014 cited by 3 of 4 papers — foundational NLP works

### Qualitative Evaluation — 5 Unseen Papers

Scored 1–5 on: Accuracy (developer perspective), Completeness (developer), Triage Value (end-user — no prior knowledge needed).

| Paper | Accuracy | Completeness | Triage Value | Avg |
|-------|----------|--------------|--------------|-----|
| LoRA | 3 | 2 | 2 | 2.33 |
| InstructGPT | 2 | 2 | 2 | 2.00 |
| LLaMA | 3 | 3 | 3 | 3.00 |
| LLMs Can Self-Improve | 2 | 2 | 2 | 2.00 |
| RAG | 4 | 3 | 4 | 3.67 |
| **Average** | **2.80** | **2.40** | **2.60** | **2.60** |

**Key finding:** Summarization success is the single strongest predictor of triage usefulness. The RAG paper (Triage Value = 4/5) was the only qualitative paper where BART generated a complete summary, allowing a researcher to make a confident read/skip decision from the report alone.
